# Route: COMPARISON

**Fires when** the reader's task is to weigh known options against each other. This is the only route where comparison is the primary surface by default.

**Primary surface:** the matrix.

**Read alongside:** techniques/evidence.md, techniques/charts.md, techniques/visual-thesis.md; techniques/screen-allocation.md for large matrices.

## Page archetypes

| Page | Question it answers |
|---|---|
| Comparison matrix | how do these differ on the dimensions that matter |
| Frontier / scatter | which options dominate others, and on what axes |
| Profiles | what is each option like in depth |
| Constraints & elimination | which options are ruled out by a hard requirement |
| Sources & methodology | where did these numbers come from |

## Mandatory mechanics

- **Comparable values in identical visual positions.** Same dimension, same place, every option.
- **Reference-item comparison:** the reader can pin one option and see others as deltas from it.
- **Difference highlighting:** where options are the same, say so quietly; where they diverge, make it visible without horizontal memory.
- **Variants with materially different behavior get their own rows.** Never collapse a quantized variant, a reasoning level, or a trim into one product label.
- Frozen key columns on wide matrices; sorting on every quantitative column; grouping by class or tier.
- A shared comparison surface remains available even when profiles go deep. The reader never has to open five pages and remember.

## Non-comparable classes

If the options do not genuinely share axes, split into separate comparison surfaces, one per class, with an explicit callout explaining why the axes cannot be shared. Never force one matrix. Presenting fundamentally different products as one class is a recorded failure.

## Conclusion form

An opinionated ranking or verdict, traceable to visible evidence. Distinguish frontier membership from practical recommendation; they are not the same claim.

## Intermediates delta

plan.md adds: the column set with units, and the reference item. verification.md adds: confirm every column is populated or explicitly unknown for every row.

## Anti-patterns

- Pros-and-cons cards where a structured comparison would be clearer.
- An aggregate score with no visible components.
- A profile-only presentation with no shared surface.

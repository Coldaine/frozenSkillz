# Route: TOPOLOGY / ATLAS

**Fires when** the subject is a spatial or networked system worth navigating: infrastructure, codebases, ecosystems, deployments.

**Primary surface:** a map with semantic scaling.

**Read alongside:** techniques/semantic-scaling.md (mandatory), techniques/relationship-grammar.md (mandatory), techniques/screen-allocation.md, techniques/visual-thesis.md.

## Page archetypes

| Page | Question it answers |
|---|---|
| Atlas index | what regions exist and how do I get to them |
| Map | where is everything and what connects to what |
| Node profiles | what is this thing in detail |
| Failure / blast radius | what dies when this dies |
| Capacity & saturation | where is headroom, where is the bottleneck |
| Change over time | what moved recently |

## Mandatory mechanics

- Representation changes with scale (overview / operational / device / flow); shrinking alone is a failure.
- Spatial positions persist across scale changes; the reader's map survives.
- Full relationship grammar: medium, capacity, certainty, direction, relevance.
- Selection dims rather than deletes context; a traced path stays legible against a dimmed whole.
- Aggregation at distance with retained summary badges, splitting into members as focus allows.
- Layout stable across views; transitions animated enough to preserve the mapping.

## Considerations

- Operational stakes deserve their own view: a topology with no failure view answers "what is there" but not "what should I worry about."
- Real inventories are heterogeneous; do not force one node representation onto materially different device classes.
- Where the map is generated from a live source, state the collection time; a stale map is a wrong map.

## Conclusion form

Structural risks; bottlenecks; single points of failure; growth paths.

## Intermediates delta

plan.md: the scale ladder (what each level shows), the node taxonomy, and the channel assignments. dossier.md: entries for edges as well as nodes, since capacity and medium claims need sources too.

## Anti-patterns

- A hairball presented as a map.
- Zoom that magnifies without revealing.
- One representation for every node class.
